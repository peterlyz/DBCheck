📖 API 调用示例
curl 命令示例
替换 YOUR_API_KEY 为实际 Key：
健康检查
curl http://localhost:5003/api/v1/health
触发巡检（同步等待结果）
curl -X POST http://localhost:5003/api/v1/inspect \
  -H "X-API-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"db_type":"mysql","host":"192.168.1.100","port":3306,"user":"root","password":"****"}'
异步巡检 + 查询结果
# 1. 触发（返回 task_id）
curl -X POST http://localhost:5003/api/v1/inspect \
  -H "X-API-Key: YOUR_API_KEY" -H "Content-Type: application/json" \
  -d '{"db_type":"oracle","host":"192.168.1.200","service_name":"ORCL","user":"system","password":"****","mode":"async"}'

# 2. 查询结果
curl -H "X-API-Key: YOUR_API_KEY" http://localhost:5003/api/v1/inspect/{task_id}
SSH 跳板巡检
curl -X POST http://localhost:5003/api/v1/inspect \
  -H "X-API-Key: YOUR_API_KEY" -H "Content-Type: application/json" \
  -d '{"db_type":"oracle","host":"192.168.1.200","port":1521,"service_name":"ORCL","user":"system","password":"****","ssh":{"host":"10.0.0.1","user":"oracle","password":"ssh_pass"}}'
支持的 db_type：mysql, pg, oracle, dm, sqlserver, tidb, ivorysql, yashandb  |  可选参数：database, sysdba, inspector, timeout
⚠️ 安全建议：生产环境请使用 HTTPS 反向代理（nginx）+ 内网部署，或使用 SSH 隧道传输。避免在公网传输明文密码。

🤖 AI 诊断设置
开启 AI 诊断后，每次巡检完成后将自动把关键指标发给大语言模型，生成个性化优化建议，附加在报告中。
注意：AI 诊断仅发送指标数值，不发送密码等敏感信息。
默认关闭，数据仅在本地处理。开启后可调用 OpenAI / DeepSeek 等远程 API，数据将通过网络发送至第三方
💡 DeepSeek 用法：API URL 填 https://api.deepseek.com/v1，模型填 deepseek-chat，API Key 填 DeepSeek 的 key
建议 300~600 秒，大模型加载较慢

🔧 数据库驱动设置
配置数据库客户端库，用于连接需要本地驱动的数据库（Oracle、YashanDB）。
SQL Server 需要安装 Microsoft ODBC Driver（内置安装包可供使用）。
注意：修改后需要重启 DBCheck 才能生效。
⬇️ 驱动自动配置
一键下载并解压数据库客户端库到 DBCheck 目录，无需手动前往官网下载。
当前下载版本：Oracle Instant Client 23.26，支持 Windows / Linux / macOS。
⚠️ Linux 用户注意：YashanDB Linux 驱动下载后为 .tar.gz 压缩包，需要手动解压到 drivers/yashandb/lib 目录。
✏️ 手动配置（高级）
手动指定 Oracle Instant Client 路径，适用于已自行下载客户端库的场景。
Windows 示例：C:\oracle\instantclient_21_12 Linux 示例：/usr/lib/oracle/21/client64/lib 留空则自动检测系统环境变量
🖥️ 服务器巡检
支持本机巡检（无需 SSH）和远程服务器巡检（通过 SSH 连接）。
